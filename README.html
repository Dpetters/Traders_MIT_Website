Version 0.1.1 Traders@MIT Website INSTALLATION IMPORTANT: If you are setting this up on windows, read the "WINDOWS INSTALLATION PREP" section below first! 1. Create a python virtual environment using virtualenv named TRADERS by running “virtualenv TRADERS” 2. For convenience, create the following aliases in "~/.bash_profile" (unix) or in "C:\bin\cmd_aliases.txt" (windows). Note that for unix you need the single quotes around the commands. For windows you don't. alias td='cd $TRADERS_AT_MIT_WEBSITE alias ts='source $TRADERS_AT_MIT_VIRTUALENV/bin/activate where $TRADERS_AT_MIT_VIRTUALENV is the path to the virtual environment that you created in step 1. $TRADERS_AT_MIT_WEBSITE is the path to the project root. Try opening a new cmd and running “td” followed by “ts”. This should activate the environment and put you in the right directory. 2. Download the winpy32 executable from http://www.lfd.uci.edu/~gohlke/pythonlibs/. Activate your environment by running "ts". Now run "pip install $EXECUTABLE" where $EXECUTABLE is the path to the .exe u just downloaded. 3. Once you have the environment activated (by running “ts”), run "pip install –r requirements.txt" from the project root. 4. Create a copy of settings_local_template.py in the project root and name it settings_local.py 5. Open settings_local.py and replace """NAME, EMAIL""" with your name and email in ADMINS. 10. Run “python manage.py syncdb --migrate”. 11. Run “fab load_data” 12. Go to localhost:8000. You’re all set! WINDOWS INSTALLATION PREP 1. If you do not have git installed, then install msysgit and add its bin as well as the mingw's bin folders to the path. Create a HOME environment variable pointing to home directory. 2. Install setuptools 3. Install virtualenv 4. Create '/bin' directory in C:. TYPICAL WORKFLOW 1. Open two terminals. In the first one run - "ts", "td", "python manage.py runserver". Leave the second for making commits. 3. Make whatever changes you need to. 4. Run "git add -A" or "git add " to stage the files you changed. 5. Run "git commit -m ''" to commit your changes. 6. Run "git push" to push the changes to staging. 7. Run "fab update" locally to update staging.
